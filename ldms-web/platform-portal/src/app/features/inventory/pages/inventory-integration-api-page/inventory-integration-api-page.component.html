sync
Who owns what?

With Sync via API, your ERP remains the system of record for master data and opening stock. LDMS owns operational workflows — reservations, transfers with en-route stops, GRVs, shipments, and billing — using the data you push. Configure outbound webhooks on Inventory integration setup so LDMS changes flow back to your ERP.

rocket_launch

Get live in three steps

Push dispatches from your ERP or WMS to LDMS — the platform handles GRV, billing, and shipment creation automatically.

  1. 1
    Generate an API key

    Go to Cross-dock integration setup and create a key for your external system.

  2. 2
    POST a dispatch ingest

    When your ERP confirms a dispatch, POST the order reference, customer, warehouse, and line items to the ingest endpoint. Use your SKU codes or LDMS product IDs.

  3. 3
    Receive the GRV callback

    When goods are received, LDMS posts a webhook to your registered callback URL with the GRV reference, matched items, and delivery timestamps.

webhook

GRV callback webhook

When goods are received (GRV created), LDMS sends a POST to your registered callback URL.

POST → Your URL https://your-system.com/ldms/grv-callback

Register your callback URL on Cross-dock integration setup. LDMS retries up to 3 times with exponential back-off on non-2xx responses.

Callback payload

{{ '{' }}
  "event": "grv.created",
  "dispatchRef": "DISP-20260617-001",
  "grvId": 10842,
  "grvNumber": "GRV-20260617-0042",
  "warehouseId": 1,
  "receivedAt": "2026-06-17T14:30:00",
  "items": [
    {{ '{' }}

    {{ '}' }}

{{ '}' }}
security
Signature verification

Each POST includes an X-LDMS-Signature header: HMAC-SHA256(secret, raw_body). Your callback secret is configured on Inventory integration setup. Verify the signature before processing.
